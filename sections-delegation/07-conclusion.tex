\section{Conclusion}

Delegation assurance names an evidence standard for the claim that a machine action remained within delegated authority. That standard matters because AI systems increasingly act through language, tools, memory, and evaluators whose boundaries are easy to collapse.

The core contribution is a translation: familiar AI-security failures can be redescribed as failures of authority uptake, action licensing, information flow, persistence, audit reconstruction, or evaluation. That translation is useful only if it remains bounded. The framework applies when a system exercises delegated action, access, disclosure, persistence, judgment, evaluation, or record-keeping, not when an AI failure concerns only a no-tool output or model capability in isolation.

The resulting paper is narrow by design. It doesn't ask reviewers to accept delegation assurance as a new threat category. It asks whether agent-security systems can support a reviewable claim about authority, scope, action, evidence, and attribution. The framework earns its keep only when it changes what evaluators and deployers record: not just whether the final answer looked safe, but whether a third party can reconstruct who licensed the action, what limits applied, how language-mediated authority was interpreted, which tool gates were crossed, and which accountable actor remains answerable.
