\section{Conclusion}

Delegation assurance names an evidence standard for the claim that a machine action remained within delegated authority. That standard matters because AI systems increasingly act through language, tools, memory, and evaluators whose boundaries are easy to collapse.

The core contribution is a translation: familiar AI-security failures can be redescribed as failures of authority uptake, action licensing, information flow, persistence, audit reconstruction, or evaluation. That translation is useful only if it remains bounded. Ordinary no-tool hallucination, benchmark contamination, model extraction, infrastructure compromise, and generic capability shortfall aren't central delegation failures unless they alter authorized action, action evidence, or accountability records.

The next version of the paper should do three things. First, it should fill in the existing-work section with the verified agent-security, audit-protocol, and safety-case literature. Second, it should develop the two tables into polished evaluation requirements. Third, it should add one running example that follows a delegated action from instruction through tool use, audit record, and judge validation. The target is a narrow paper: delegation assurance as an evidentiary grammar for justified machine action.
