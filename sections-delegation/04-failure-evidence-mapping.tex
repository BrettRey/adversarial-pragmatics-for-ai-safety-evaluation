The corresponding evaluation requirements are shown in Table~\ref{tab:delegation-evidence}. A delegation-assurance evaluation doesn't need every row for every deployment. The rows are evidence families. A calendar assistant, a coding agent, a medical summarization system, and an institutional triage tool will weight them differently. Each safety claim still has to name the evidence family it relies on and the test that could defeat it.

The procurement example shows the unit of evaluation. A final memo can be correct while the trace still fails delegation assurance if the system used an unauthorized email tool, disclosed the budget ceiling, or updated a vendor record without permission. Conversely, a refusal can be safe at the output layer and still fail delegation assurance if the assistant abandoned a licensed comparison task because it encountered quoted unsafe text. The target is the justified action claim, not the final answer alone.
